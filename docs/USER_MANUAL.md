## 系統簡介

禾一系統出勤系統是一套完整的企業考勤管理解決方案，提供以下核心功能：

- **打卡管理**：QR Code 掃描打卡、GPS 定位驗證
- **請假管理**：線上申請、審批流程、假別額度追蹤
- **加班管理**：加班申請、補休/加班費選擇
- **補打卡**：忘記打卡時的補救機制
- **報表系統**：出勤摘要、異常清單、特休資格計算
- **通知系統**：即時通知審批結果與系統訊息
- **主管功能**：部門出勤總覽、批次審批
- **HR 管理**：員工管理、假別額度設定

### 系統需求

- 瀏覽器：Chrome、Firefox、Safari、Edge（最新版本）
- 需開啟定位權限（用於 GPS 打卡驗證）
- 需開啟相機權限（用於 QR Code 掃描）

## 登入與帳號管理

### 登入系統

1. 開啟系統網址
2. 輸入**員工編號**（帳號）
3. 輸入**密碼**
4. 點擊「登入」按鈕

![登入頁面示意]

### 忘記密碼

1. 在登入頁面點擊「忘記密碼？」
2. 輸入您註冊時使用的 Email
3. 點擊「發送臨時密碼」
4. 至信箱收取臨時密碼
5. 使用臨時密碼登入後，建議立即修改密碼

### 修改密碼

1. 登入後，點擊右上角「修改密碼」按鈕
2. 輸入**舊密碼**
3. 輸入**新密碼**（至少 8 個字元）
4. 再次輸入新密碼確認
5. 點擊「確認修改」

### 登出

點擊右上角「登出」按鈕即可安全登出系統。

## 一般員工功能

### 4.1 打卡

#### QR Code 掃描打卡

1. 點擊首頁的「📷 掃描打卡」按鈕
2. 允許瀏覽器存取相機
3. 將相機對準公司提供的打卡 QR Code
4. 系統自動辨識並完成打卡
5. 畫面顯示打卡成功訊息

**注意事項：**
- 系統會記錄您的 GPS 位置
- 請確保在公司允許的打卡範圍內
- 上班打卡後需再進行下班打卡

#### 查看打卡記錄

首頁下方「最近打卡記錄」區塊顯示最近 7 天的打卡紀錄，包含：
- 日期
- 上班/下班時間
- 工時
- 遲到/早退標記
- 補打卡標記

---

### 4.2 請假管理

#### 申請請假

1. 點擊首頁的「📝 申請請假」按鈕
2. 填寫請假表單：
   - **假別**：選擇假別類型（特休、病假、事假等）
   - **開始時間**：選擇請假開始日期與時間
   - **結束時間**：選擇請假結束日期與時間
   - **請假時數**：系統自動計算，可手動調整
   - **請假原因**：填寫請假事由
   - **職務代理人**：選擇代理人（選填）
   - **附件**：上傳證明文件（選填）
3. 點擊「送出申請」

#### 查看請假記錄

首頁「我的請假記錄」區塊顯示請假紀錄，可使用篩選條件：
- **時間範圍**：最近 30/60/90 天或本年度
- **狀態**：全部、待審批、已批准、已拒絕、已取消

#### 查看假別額度

首頁「假別額度」區塊顯示各假別的：
- 總額度
- 已使用時數
- 剩餘時數

---

### 4.3 加班管理

#### 申請加班

1. 點擊首頁的「⏰ 申請加班」按鈕
2. 填寫加班表單：
   - **加班日期**：選擇加班日期
   - **開始時間**：輸入加班開始時間
   - **結束時間**：輸入加班結束時間
   - **加班時數**：系統自動計算
   - **補償方式**：選擇「補休」或「加班費」
   - **加班原因**：填寫加班事由
3. 點擊「送出申請」

#### 查看加班記錄

首頁「我的加班記錄」區塊顯示加班紀錄，包含：
- 加班日期
- 加班時段
- 加班時數
- 補償方式
- 審批狀態

#### 取消加班申請

若申請尚在「待審批」狀態，可點擊「取消」按鈕撤回申請。

---

### 4.4 補打卡

#### 申請補打卡

1. 點擊首頁的「🔧 補打卡」按鈕
2. 填寫補打卡表單：
   - **補打卡類型**：上班打卡或下班打卡
   - **補打卡日期**：選擇日期
   - **補打卡時間**：輸入實際應打卡時間
   - **補打卡原因**：填寫原因（如：忘記打卡、系統故障等）
3. 點擊「送出申請」

**注意事項：**
- 每月補打卡次數有限制（顯示於按鈕上）
- 超過限制次數將無法申請

#### 查看補打卡記錄

首頁「我的補打卡申請」區塊顯示申請記錄，可切換檢視：
- 全部
- 待審批
- 已批准
- 已拒絕

---

### 4.5 報表查看

點擊首頁的「📊 查看報表」按鈕開啟報表中心。

#### 出勤摘要

顯示指定月份的出勤統計：
- 應出勤天數
- 實際出勤天數
- 遲到次數/分鐘數
- 早退次數/分鐘數
- 請假天數
- 加班時數

#### 異常清單

列出需要關注的出勤異常：
- 遲到記錄
- 早退記錄
- 缺卡記錄
- 超時加班

#### 特休資格

根據到職日期計算特休天數：
- 顯示年資
- 當年度應有特休
- 已使用天數
- 剩餘可用天數

#### 假別額度

詳細顯示各假別額度使用情況。

---

### 4.6 通知系統

#### 查看通知

點擊右上角的「🔔」通知鈴鐺圖示，顯示：
- 未讀通知數量（紅色徽章）
- 通知列表（最新在前）

通知類型包括：
- 審批結果通知（請假/加班/補打卡）
- 系統公告
- 提醒訊息

#### 標記已讀

- 點擊單一通知可標記為已讀
- 點擊「全部標記已讀」可一次清除所有未讀

## 主管功能

**適用角色：** 部門主管、HR 管理員、總經理

登入後，頁面頂部會顯示「主管功能」按鈕。

### 5.1 出勤總覽

查看部門員工的當日出勤狀況：
- 已出勤人數
- 請假人數
- 未出勤人數
- 異常人數

### 5.2 批次審批

快速處理多筆待審批申請。

#### 操作步驟

1. 點擊「主管功能」→「批次審批」
2. 選擇審批類型標籤：
   - **請假**：顯示待審批的請假申請
   - **加班**：顯示待審批的加班申請
   - **補打卡**：顯示待審批的補打卡申請
3. 勾選要處理的申請（可多選）
4. 點擊「批准選取」或「拒絕選取」
5. 輸入審批意見（選填）
6. 確認送出

### 5.3 部門報表

查看部門整體出勤統計報表：
- 選擇年份與月份
- 查看部門出勤率
- 各員工出勤統計
- 異常排行

## HR 管理功能

**適用角色：** HR 管理員、總經理

登入後，頁面頂部會顯示「HR 管理」按鈕。

### 6.1 員工管理

#### 查看員工列表

1. 點擊「HR 管理」開啟員工管理介面
2. 使用篩選條件：
   - **搜尋**：輸入員工編號或姓名
   - **角色**：篩選特定角色
   - **狀態**：在職/離職/全部
3. 點擊「搜尋」查看結果

#### 新增員工

1. 點擊「+ 新增員工」按鈕
2. 填寫員工資料：
   - 員工編號（必填，唯一）
   - 姓名（必填）
   - Email（必填）
   - 電話
   - 角色
   - 部門
   - 到職日期
3. 點擊「儲存」

#### 編輯員工

1. 在員工列表找到目標員工
2. 點擊「編輯」按鈕
3. 修改需要更新的欄位
4. 點擊「儲存」

### 6.2 假別額度設定

HR 可批次設定員工的假別額度：

1. 在員工管理介面選擇員工
2. 設定假別類型與額度
3. 批次套用到選取的員工

## 資料匯出

**適用角色：** 部門主管、HR 管理員、總經理

### 匯出步驟

1. 點擊頁面頂部的「匯出」按鈕
2. 選擇匯出設定：
   - **匯出類型**：出勤記錄 / 請假記錄
   - **匯出格式**：CSV / Excel (XLSX)
   - **日期範圍**：選擇開始與結束日期
3. 使用快速選擇按鈕：
   - 本月
   - 上月
   - 今年
4. 點擊「開始匯出」
5. 檔案自動下載

### 匯出欄位說明

#### 出勤記錄匯出

| 欄位 | 說明 |

## 常見問題

### Q1: 忘記打卡怎麼辦？

使用「補打卡」功能申請，填寫正確的打卡時間與原因，送出後等待主管審批。

### Q2: 為什麼無法使用 QR Code 打卡？

請確認：
1. 瀏覽器已允許相機權限
2. 瀏覽器已允許定位權限
3. 您在公司允許的打卡範圍內
4. QR Code 清晰可辨識

### Q3: 請假申請被拒絕了，可以重新申請嗎？

可以。被拒絕的申請不會影響您的假別額度，您可以修改內容後重新送出申請。

### Q4: 加班補休如何使用？

加班申請選擇「補休」作為補償方式，審批通過後，補休時數會自動加入您的補休額度，之後可用於請「補休假」。

### Q5: 為什麼我看不到「主管功能」或「HR 管理」按鈕？

這些按鈕僅對具有相應權限的角色顯示。如果您認為應該有此權限，請聯繫 HR 或系統管理員確認您的角色設定。

### Q6: 匯出的 Excel 檔案打開是亂碼？

請確認使用 Microsoft Excel 或 LibreOffice Calc 開啟。如果仍有問題，嘗試：
1. 開啟 Excel
2. 選擇「資料」→「從文字/CSV」
3. 選擇匯出的檔案
4. 編碼選擇「UTF-8」

### Q7: 系統顯示「GPS 位置未取得」？

請確認：
1. 瀏覽器已允許定位權限
2. 裝置的定位功能已開啟
3. 嘗試重新整理頁面

### Q8: 如何聯繫系統管理員？

請聯繫您公司的 HR 部門或 IT 支援團隊。
